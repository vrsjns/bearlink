# Move OutboxPoller to shared

> **Status:** draft
> **Service(s):** shared, auth-service, url-service
> **Priority:** low

## Goal

The outbox poller is duplicated verbatim between auth-service and url-service,
differing only by the hardcoded `sourceService` string. Consolidating it into
`shared/` eliminates duplication, ensures bug fixes and behaviour changes apply
everywhere at once, and makes it trivially easy for any future service to adopt
the outbox pattern.

## Background

Both `auth-service/services/outboxPoller.js` and
`url-service/services/outboxPoller.js` implement `createOutboxPoller({ prisma,
auditServiceUrl, logger })`. The two files are identical except for line 37
where the serialised batch hardcodes `sourceService: 'auth-service'` or
`sourceService: 'url-service'`. The poller is not yet in `shared/` because it
was implemented per-service before the pattern was established.

## Requirements

### Functional

- **R1:** `shared/` shall expose `createOutboxPoller({ prisma, auditServiceUrl,
logger, sourceService })` with identical runtime behaviour to the existing
  per-service implementations.
- **R2:** `auth-service` shall remove its local `outboxPoller.js` and import
  `createOutboxPoller` from `shared/`, passing `sourceService: 'auth-service'`.
- **R3:** `url-service` shall remove its local `outboxPoller.js` and import
  `createOutboxPoller` from `shared/`, passing `sourceService: 'url-service'`.
- **R4:** `shared/index.js` shall export `createOutboxPoller` so services can
  reach it via the package's main entry point.

### Non-Functional

- **R5:** No new npm dependencies shall be introduced.
- **R6:** The shared implementation shall be covered by a unit test suite in
  `shared/` that verifies all observable behaviours: batching, HTTP delivery,
  `sourceService` forwarding, mark-as-processed on 2xx, retry on non-2xx,
  and clean shutdown.
- **R7:** Outbox poller test files in url-service and auth-service shall be
  deleted. Consumers have no testing responsibility for a shared library.

## Acceptance Criteria

- [ ] `shared/utils/outboxPoller.js` (or equivalent path) exists and exports
      `createOutboxPoller`.
- [ ] `auth-service/services/outboxPoller.js` is deleted.
- [ ] `url-service/services/outboxPoller.js` is deleted.
- [ ] `shared/index.js` exports `createOutboxPoller`.
- [ ] Given a batch of unprocessed outbox rows, when the poller fires for
      auth-service, the POSTed payload contains `sourceService: 'auth-service'`
      for every row.
- [ ] Given a batch of unprocessed outbox rows, when the poller fires for
      url-service, the POSTed payload contains `sourceService: 'url-service'`
      for every row.
- [ ] `shared/` contains a unit test file for `createOutboxPoller` covering:
      batching, successful delivery (rows marked processed), non-2xx retry
      (rows left unprocessed), timeout/abort, `sourceService` forwarding, and
      clean stop.
- [ ] url-service outboxPoller test file is deleted.
- [ ] `npm test` passes across all workspaces.

## Out of Scope

- Changing the polling interval, batch size, or retry strategy.
- Adding new outbox events or modifying the audit-service receiver.
- Wiring the outbox pattern into analytics-service, audit-service, or any other
  service not currently using it.
